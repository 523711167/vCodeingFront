# AGENTS.md

## Working agreements

- 以后在这个仓库中新增或修改任何代码文件时，都必须补充清晰注释。
- 注释要求覆盖所有类型文件，包括但不限于：`TypeScript`、`JavaScript`、`CSS`、`HTML`、配置文件、脚本文件、Mock 文件。
- 注释重点说明三类信息：
  - 该段代码的职责是什么。
  - 为什么采用当前实现方式。
  - 后续如果要扩展，应当从哪里接入。
- 不允许只写无信息量注释，例如“定义变量”“设置值”“渲染组件”。
- 优先在这些位置补注释：
  - 应用入口、路由、状态管理、请求层、权限逻辑。
  - 公共组件、样式规则、环境配置、构建配置。
  - 业务页面中的关键数据流、表单行为、列表交互、异常处理。
- 当文件内容较简单时，也要至少补充文件级或关键逻辑级注释，不能完全无注释。
- 修改旧代码时，如果发现周边关键逻辑没有注释，应顺手补齐，保持风格一致。
